# 代码规范

- 采用 PEP8 规范
- 注释采用中文，注释内容尽量只占一行，保持简明扼要的注释风格
- 不要重复造轮子，优先使用common目录下的工具函数、异常定义等

# 协作模式

## 设计讨论模式

**触发词**: "我们先聊一下", "我们讨论一下"

**含义**: 用户希望就具体需求的实现设计进行讨论

**AI 行为准则**:
1. 暂停编码，进入讨论模式
2. 主动询问需求细节和约束条件
3. 提出可行的技术方案供选择
4. 与用户共同完善设计
5. 待用户确认设计后，编写需求设计文档，放在docs/reqs下，文件名格式为YYMMDD-xxx.md，其中YYMMDD为当前日期，xxx为需求描述
6. 获得用户许可后，方可开始编码

**禁止事项**:
- 直接生成代码
- 未经讨论就假设实现方案
