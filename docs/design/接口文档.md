# 接口文档

本文档描述 Genban 服务端提供的 HTTP 接口。

---

## 用户模块

### GET /api/user/key

获取 RSA 公钥，用于前端加密密码。

#### 响应参数

| 字段       | 类型   | 说明                     |
| ---------- | ------ | ------------------------ |
| public_key | string | RSA 公钥内容（PEM 格式） |
| error      | string | 错误信息，成功时为空     |

#### 响应示例（成功）

```json
{
  "public_key": "-----BEGIN PUBLIC KEY-----\nMIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA...\n-----END PUBLIC KEY-----",
  "error": ""
}
```

#### 响应示例（失败）

```json
{
  "public_key": "",
  "error": "公钥文件不存在"
}
```

#### 调用时机

前端应在用户输入密码前调用此接口获取公钥，用于加密密码后再调用 `/api/user/login`。

### POST /api/user/login

用户登录/自动注册接口。若用户不存在则自动注册，存在则验证登录。

#### 请求参数

| 字段     | 类型   | 必填 | 说明                                  |
| -------- | ------ | ---- | ------------------------------------- |
| user_id  | string | 是   | 用户 ID                               |
| password | string | 是   | 密码（RSA 加密后的密文，Base64 编码） |

#### 请求示例

```json
{
  "user_id": "user001",
  "password": "RSA加密后的密文..."
}
```

#### 密码加密说明

1. 调用 `/api/user/key` 获取 RSA 公钥
2. 前端使用 RSA 公钥对密码进行加密
3. 加密后的二进制数据使用 Base64 编码
4. 将 Base64 字符串作为 password 字段的值

#### 密码强度要求

- 最少 6 位字符
- 注册时若密码强度不足会返回错误

#### 响应参数

| 字段       | 类型    | 说明                     |
| ---------- | ------- | ------------------------ |
| token      | string  | 登录令牌，后续请求需携带 |
| expires_at | integer | 令牌过期时间戳（秒）     |
| error      | string  | 错误信息，成功时为空     |

#### 响应示例（成功）

```json
{
  "token": "x9aBcDeFgHiJkLmNoPqRsTuVwXyZ...",
  "expires_at": 1712345678,
  "error": ""
}
```

#### 响应示例（失败）

```json
{
  "token": "",
  "expires_at": 0,
  "error": "密码最少需要6位字符"
}
```

#### 错误码说明

| 错误信息              | 说明                 |
| --------------------- | -------------------- |
| 密码最少需要 6 位字符 | 注册时密码强度不足   |
| 密码错误              | 登录时密码验证失败   |
| RSA 私钥未配置        | 服务端未配置私钥文件 |
| 用户创建失败          | 数据库写入失败       |
| 用户状态初始化失败    | 用户状态创建失败     |
| 用户名只能包含字母、数字、下划线 | 注册时用户名格式不合法 |

## Assistant 模块

### POST /api/assistant/submit

提交用户消息到处理队列。消息将被放入异步队列，由 AgentWorker 在独立线程中处理。

#### 请求头

| 字段          | 必填 | 说明                               |
| ------------- | ---- | ---------------------------------- |
| Authorization | 是   | Bearer <token>，登录接口获取的令牌 |

#### 请求参数

| 字段    | 类型   | 必填 | 说明               |
| ------- | ------ | ---- | ------------------ |
| message | string | 是   | 用户输入的消息内容 |

#### 请求示例

```json
{
  "message": "今天天气怎么样？"
}
```

#### 响应参数

| 字段    | 类型    | 说明                               |
| ------- | ------- | ---------------------------------- |
| success | boolean | 是否成功放入队列                   |
| chat_id | string  | 本次输入新增的 Chat ID，失败时为空 |
| error   | string  | 错误信息，成功时为空               |

#### 响应示例（成功）

```json
{
  "success": true,
  "chat_id": "171234567800010001",
  "error": ""
}
```

#### 响应示例（失败-未鉴权）

```json
{
  "success": false,
  "chat_id": "",
  "error": "未授权，请重新登录"
}
```

### GET /api/assistant/stream

建立 SSE 连接，持续接收 Agent 产生的新消息。

#### 请求头

| 字段          | 必填 | 说明                               |
| ------------- | ---- | ---------------------------------- |
| Authorization | 是   | Bearer <token>，登录接口获取的令牌 |

#### SSE 事件类型

| 事件类型 | 说明 |
| -------- | ---- |
| message  | 普通对话消息（user/assistant/tool/error 类型） |
| complete | 一次大模型调用完成信号，表示本次大模型输出完成 |
| close    | 后端无消息通知，前端收到后应主动断开连接，随后再重连 |

#### SSE 数据格式（event: message）

| 字段              | 类型    | 说明                                    |
| ----------------- | ------- | --------------------------------------- |
| id                | string  | 消息唯一 ID                             |
| type              | string  | 消息类型：user/assistant/tool/error/stop |
| time              | integer | 消息时间戳（秒）                        |
| content           | array   | 消息内容，统一为列表格式                |
| reasoning_content | string  | 推理内容（仅 assistant 类型可能有）     |
| tool_calls        | array   | 工具调用信息（仅 assistant 类型可能有） |
| channel_type      | string  | 消息渠道类型，如 `web`、`qq` 等（仅 user 类型可能有） |

**content 字段格式说明：**
- 纯文本内容：`[{"text": "消息内容"}]`
- 多模态内容：`[{"image": "http://xxx.jpg"}, {"text": "描述"}]`
- 支持的类型包括：`text`（文本）、`image`（图片）、`audio`（音频）、`video`（视频）
- 当消息类型为 `assistant` 时，`content` 中 `text` 开头可能带有 "[no_reply]" 标签，表示该消息为静默消息

#### SSE 数据示例

```
event: message
data: {"type": "user", "time": 1712345678, "id": "msg_001", "content": [{"text": "查询北京天气"}], "reasoning_content": "", "tool_calls": null}

event: message
data: {"type": "assistant", "time": 1712345678, "id": "msg_002", "content": [], "reasoning_content": "用户想查询天气", "tool_calls": null}

event: message
data: {"type": "assistant", "time": 1712345678, "id": "msg_002", "content": [{"text": "我来为您查询天气"}], "reasoning_content": "用户想查询天气，我需要调用天气工具", "tool_calls": [{"function": {"name": "get_weather", "arguments": "{\"location\": \"北京\"}"}}]}

event: message
data: {"type": "tool", "time": 1712345678, "id": "msg_003", "content": [{"text": "北京今天晴，25°C"}], "reasoning_content": "", "tool_calls": null}

event: message
data: {"type": "assistant", "time": 1712345678, "id": "msg_004", "content": [], "reasoning_content": "工具返回了天气信息", "tool_calls": null}

event: message
data: {"type": "assistant", "time": 1712345678, "id": "msg_004", "content": [{"text": "北京今天天气晴朗，气温 25°C"}], "reasoning_content": "工具返回了天气信息", "tool_calls": null}

```

#### 多模态消息示例

```
event: message
data: {"type": "user", "time": 1712345678, "id": "msg_005", "content": [{"image": "https://example.com/image.jpg"}, {"text": "这张图片里有什么？"}], "reasoning_content": "", "tool_calls": null}

event: message
data: {"type": "assistant", "time": 1712345678, "id": "msg_006", "content": [{"text": "这张图片显示了一只猫在沙发上睡觉。"}], "reasoning_content": "用户询问图片内容", "tool_calls": null}

```

#### SSE 完成事件（event: complete）

当一次大模型调用完成时，后端会向前端发送完成事件。

**事件格式：**

```
event: complete
data: {}

```

#### SSE 关闭事件（event: close）

当后端长时间没有拉取到新消息时，会向前端发送关闭事件，前端收到后应主动断开连接。

**事件格式：**

```
event: close
data: {}

```

#### 注意事项

1. **连接时机**：建议在登录成功后立即建立 SSE 连接，确保能接收到所有消息
2. **连接保持**：SSE 连接会一直保持，直到客户端主动断开或网络异常
3. **消息顺序**：消息按时间顺序推送，id 字段可用于排序
4. **错误处理**：网络异常时会触发 onerror，前端应实现重连机制
5. **关闭事件**：收到 `event: close` 时应主动关闭连接，然后在较短时间后重新连接以获取最新消息
6. **完成事件**：`complete` 事件表示一次大模型输出的完成，前端应在收到此事件后进行相应的 UI 状态更新

### POST /api/assistant/stop

停止当前对话。向 Agent 发送停止信号，中断当前正在进行的对话处理。

#### 请求头

| 字段          | 必填 | 说明                               |
| ------------- | ---- | ---------------------------------- |
| Authorization | 是   | Bearer <token>，登录接口获取的令牌 |

#### 响应参数

| 字段    | 类型    | 说明                      |
| ------- | ------- | ------------------------- |
| success | boolean | 是否成功                  |
| chat_id | string  | 本次停止消息的 Chat ID    |
| error   | string  | 错误信息                  |

#### 响应示例（成功）

```json
{
  "success": true,
  "chat_id": "171234567800010001",
  "error": ""
}
```

#### 响应示例（失败-未鉴权）

```json
{
  "success": false,
  "chat_id": "",
  "error": "未授权，请重新登录"
}
```

#### 使用场景

1. **用户主动停止**：用户点击"停止生成"按钮时调用
2. **切换对话**：切换对话前停止当前对话
3. **页面关闭**：页面关闭前调用以释放资源

### POST /api/assistant/history

查询用户历史对话内容。以 chat_id 为起点往前查询 count 条历史对话，若 chat_id 为空则返回最新的 count 条。

#### 请求头

| 字段          | 必填 | 说明                               |
| ------------- | ---- | ---------------------------------- |
| Authorization | 是   | Bearer <token>，登录接口获取的令牌 |
| Content-Type  | 是   | application/json                   |

#### 请求参数

| 字段      | 类型    | 必填 | 默认值 | 说明                                           |
| --------- | ------- | ---- | ------ | ---------------------------------------------- |
| chat_id   | string  | 否   | null   | 查询起点 chat ID，不传则返回最新的 count 条    |
| chat_time | integer | 否   | null   | 查询起点时间戳（秒级），用于路由文件           |
| count     | integer | 否   | 20     | 查询数量，范围 1-100                           |

#### 请求示例

```json
{
  "chat_id": "81",
  "chat_time": 1712345678,
  "count": 20
}
```

#### 响应参数

| 字段    | 类型    | 说明                                           |
| ------- | ------- | ---------------------------------------------- |
| success | boolean | 查询是否成功                                   |
| chats   | array   | 历史对话列表（按时间升序排列，从旧到新）       |
| total   | integer | 返回的对话数量                                 |
| error   | string  | 错误信息（success 为 false 时返回）            |

**chats 中每个对象的字段：**

| 字段              | 类型    | 说明                                           |
| ----------------- | ------- | ---------------------------------------------- |
| id                | string  | Chat 唯一标识                                  |
| type              | string  | Chat 类型：user/assistant/tool/error/stop      |
| time              | integer | Chat 时间戳（秒级）                            |
| content           | array   | 内容列表，格式为 `[{"text": "内容"}]`          |
| reasoning_content | string  | 推理内容（如有）                               |
| tool_calls        | array   | 工具调用信息（如有）                           |
| channel_type      | string  | 消息渠道类型，如 `web`、`qq` 等（仅 user 类型可能有） |

#### 响应示例（成功）

```json
{
  "success": true,
  "chats": [
    {
      "id": "61",
      "type": "user",
      "time": 1712345600,
      "content": [{"text": "你好"}],
      "reasoning_content": "",
      "tool_calls": null
    },
    {
      "id": "62",
      "type": "assistant",
      "time": 1712345605,
      "content": [{"text": "你好！有什么可以帮助你的吗？"}],
      "reasoning_content": "",
      "tool_calls": null
    }
  ],
  "total": 2,
  "error": ""
}
```

#### 响应示例（失败）

```json
{
  "success": false,
  "chats": [],
  "total": 0,
  "error": "历史对话查询失败"
}
```

#### 查询逻辑说明

1. **指定 chat_id 查询**：以 `chat_id` 为起点，往前查询 `count` 条历史对话（不包含该 `chat_id`）

   示例：历史 chat ID 为 1-100，传入 `chat_id=81`、`count=20`，返回 ID 为 61-80 的 chat

2. **不指定 chat_id 查询**：返回最新的 `count` 条历史对话

3. **返回顺序**：响应中的 `chats` 数组按时间升序排列（从旧到新）

4. **参数限制**：`count` 参数若小于 1 则按 1 处理，大于 100 则按 100 处理

## FileSystem 模块

文件系统模块提供文件操作相关的 HTTP 接口，所有操作限制在用户专属目录下。

### GET /api/file_system/list

查看目录内容。

#### 请求头

| 字段          | 必填 | 说明                               |
| ------------- | ---- | ---------------------------------- |
| Authorization | 是   | Bearer <token>，登录接口获取的令牌 |

#### 请求参数

| 字段 | 类型   | 必填 | 说明                           |
| ---- | ------ | ---- | ------------------------------ |
| path | string | 否   | 相对路径，默认为用户根目录     |

#### 响应参数

| 字段  | 类型  | 说明                 |
| ----- | ----- | -------------------- |
| items | array | 目录项列表           |

items 中每个对象的字段：

| 字段        | 类型    | 说明                           |
| ----------- | ------- | ------------------------------ |
| name        | string  | 名称                           |
| type        | string  | 类型：directory/file           |
| size        | integer | 文件大小（字节），目录为 null  |
| modified_at | string  | 修改时间（ISO 8601 格式）      |

#### 响应示例（成功）

```json
{
  "items": [
    {
      "name": "documents",
      "type": "directory",
      "size": null,
      "modified_at": "2026-03-08T10:30:00"
    },
    {
      "name": "notes.txt",
      "type": "file",
      "size": 1024,
      "modified_at": "2026-03-08T11:00:00"
    }
  ]
}
```

#### 错误码说明

| HTTP 状态码 | 说明         |
| ----------- | ------------ |
| 403         | 无权访问该路径 |
| 404         | 目录不存在   |

Settings 模块提供用户配置管理相关的 HTTP 接口，用于管理用户的模型选择和深度思考开关。

| 字段            | 类型    | 说明                           |

## Skills 模块
Skills 模块提供 Skill 管理相关的 HTTP 接口，所有操作限制在用户专属的 skills 目录下。

### GET /api/skills/list

查询当前用户所有 Skills。

### GET /api/im/channels/list

#### 请求头

      {
        "field_name": "app_id",
        "field_type": "string",
        "description": "QQ机器人应用ID",
        "required": true,
        "default_value": null
      },
      {
        "field_name": "app_secret",
        "field_type": "string",
        "description": "QQ机器人应用密钥",
        "required": true,

## 通用说明

### 鉴权方式

所有需要鉴权的接口（除 `/api/user/key` 和 `/api/user/login` 外）都需要在请求头中携带 Token：

```
Authorization: Bearer <token>
```

Token 通过 `/api/user/login` 接口获取，有效期为 7 天。

### 错误响应格式

当接口返回 401 状态码时，响应体格式如下：

```json
{
  "detail": "未授权，请重新登录"
}
```

前端应在收到 401 时引导用户重新登录。
