# 接口文档

本文档描述 Genban 服务端提供的 HTTP 接口。

---

## 用户模块

### GET /api/user/key

获取RSA公钥，用于前端加密密码。

#### 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| public_key | string | RSA公钥内容（PEM格式） |
| error | string | 错误信息，成功时为空 |

#### 响应示例（成功）

```json
{
  "public_key": "-----BEGIN PUBLIC KEY-----\nMIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA...\n-----END PUBLIC KEY-----",
  "error": ""
}
```

#### 响应示例（失败）

```json
{
  "public_key": "",
  "error": "公钥文件不存在"
}
```

#### 调用时机

前端应在用户输入密码前调用此接口获取公钥，用于加密密码后再调用 `/api/user/login`。

---

### POST /api/user/login

用户登录/自动注册接口。若用户不存在则自动注册，存在则验证登录。

#### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user_id | string | 是 | 用户ID |
| password | string | 是 | 密码（RSA加密后的密文，Base64编码） |

#### 请求示例

```json
{
  "user_id": "user001",
  "password": "RSA加密后的密文..."
}
```

#### 密码加密说明

1. 调用 `/api/user/key` 获取RSA公钥
2. 前端使用RSA公钥对密码进行加密
3. 加密后的二进制数据使用Base64编码
4. 将Base64字符串作为password字段的值

**JavaScript完整示例：**
```javascript
import JSEncrypt from 'jsencrypt';

// 1. 获取公钥
const response = await fetch('/api/user/key');
const { public_key } = await response.json();

// 2. 加密密码
const encrypt = new JSEncrypt();
encrypt.setPublicKey(public_key);
const encrypted = encrypt.encrypt(password);
const base64Password = btoa(encrypted);

// 3. 登录
await fetch('/api/user/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ user_id: "user001", password: base64Password })
});
```

#### 密码强度要求

- 最少6位字符
- 注册时若密码强度不足会返回错误

#### 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| token | string | 登录令牌，后续请求需携带 |
| expires_at | integer | 令牌过期时间戳（秒） |
| error | string | 错误信息，成功时为空 |

#### 响应示例（成功）

```json
{
  "token": "x9aBcDeFgHiJkLmNoPqRsTuVwXyZ...",
  "expires_at": 1712345678,
  "error": ""
}
```

#### 响应示例（失败）

```json
{
  "token": "",
  "expires_at": 0,
  "error": "密码最少需要6位字符"
}
```

#### 错误码说明

| 错误信息 | 说明 |
|----------|------|
| 密码最少需要6位字符 | 注册时密码强度不足 |
| 密码错误 | 登录时密码验证失败 |
| RSA私钥未配置 | 服务端未配置私钥文件 |
| 用户创建失败 | 数据库写入失败 |
| 用户状态初始化失败 | 用户状态创建失败 |
