# Genban 记忆系统架构设计

## 1. 概述

### 1.1 设计目标

记忆系统让 AI 助理具备长期记忆能力，能够：
- 跨会话保持用户相关信息
- 理解对话的历史背景
- 基于用户画像提供个性化服务

### 1.2 设计原则

1. **分层存储**：不同类型的记忆采用不同的存储策略
2. **渐进抽象**：从原始对话逐步提炼为高价值信息
3. **向量检索**：支持基于语义的相似度搜索

---

## 2. 记忆类型

系统包含五种记忆类型，分为两条独立的处理流程：

**流程一：在线上下文压缩（实时）**
```
Chat (原始对话) → Conversation Memory (对话摘要)
```
用于实时维护对话上下文，支持长对话的窗口管理。

**流程二：离线记忆提取（异步）**
```
Chat (原始对话)
    ↓
Episodic Memory (情景记忆)
    ↓
    ├────→ Semantic Memory (语义记忆) ────→ Core Memory (核心记忆)
    │
    └────→ Procedural Memory (程序记忆)
```
用于从对话中异步提取和沉淀长期记忆。

**记忆派生关系**：
- Episodic Memory 提炼自 Chat（从对话中提取时间线事件）
- Semantic Memory 提炼自 Episodic Memory（从事件中提取事实）
- Procedural Memory 提炼自 Episodic Memory（从事件中总结行为规则）
- Core Memory 提炼自 Semantic Memory（从事实中总结用户画像）

| 记忆类型 | 说明 | 核心作用 |
|---------|------|---------|
| **Chat** | 原始对话数据 | 保留完整信息用于回溯 |
| **Conversation Memory** | 对话的简练概述 | 支持长对话的上下文管理 |
| **Episodic Memory** | 记录"何时何地发生了什么" | 构建用户经历的时间线 |
| **Semantic Memory** | 提取的关键事实和概念 | 构建用户知识图谱 |
| **Core Memory** | 用户画像和助手人设 | 定义个性化交互基础 |
| **Procedural Memory** | 策略规则和行为模式 | 支持条件触发的自动化 |

## 3. 系统架构

### 3.1 整体架构

```
┌─────────────────────────────────────────┐
│              接入层                      │
│   MemoryModule / MemoryHook / MemoryTool │
└─────────────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│              服务层                      │
│   MemoryBuilder / MemoryRetriever       │
└─────────────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│              存储层                      │
│   SQLite + Neo4j (图+向量) + File        │
└─────────────────────────────────────────┘
```

### 3.2 核心组件职责

| 组件 | 职责 |
|-----|------|
| **MemoryModule** | 模块入口，注册工具和钩子 |
| **MemoryHook** | 在 Agent 生命周期中拦截读写 |
| **MemoryTool** | 供 Agent 主动调用记忆能力 |
| **MemoryBuilder** | 从对话构建各类记忆 |
| **MemoryRetriever** | 检索和召回记忆 |

---

## 4. 数据流

### 4.1 写入流程

**在线流程（实时）**
```
对话主流程 → Chat → 文件存储
                ↓
         WriteMemoryHook → 对话历史存储
                ↓
         触发上下文压缩 → Conversation Memory
```

**离线流程（异步）**
```
对话主流程 → Chat
                ↓
         触发记忆构建（异步）
                ↓
         Episodic Memory
                ↓
       ┌────────┴────────┐
       ↓                 ↓
Semantic Memory    Procedural Memory
       │
       ↓
  Core Memory
```

### 4.2 读取流程

```
Agent主流程 → ReadMemoryHook → 多源检索记忆
                                    ↓
                              融合排序 → 注入上下文
```

## 7. 总结

本架构借鉴认知科学的记忆分类理论，构建了分层递进的记忆系统：

1. **原始层**：Chat 保留完整对话
2. **会话层**：Conversation Memory 提供对话摘要
3. **情景层**：Episodic Memory 记录时间线事件
4. **语义层**：Semantic Memory 沉淀知识
5. **核心层**：Core Memory 定义用户画像
6. **程序层**：Procedural Memory 存储行为规则

通过 Hook 机制与 Agent 集成，实现透明的记忆读写能力。
